# Limitations — Mandate Vacuum Governance Intelligence

> Honest documentation of the current boundaries of this system. Understanding limitations is essential for responsible use of any policy diagnostics tool.

---

## 1. Data Assumptions

### Transfer Frequency as Accountability Proxy
The Accountability Half-Life model assumes that the frequency of inter-department transfers correlates with the erosion of accountability. This is a reasonable heuristic but has not yet been empirically validated against independent accountability measurements (e.g. citizen satisfaction surveys, resolution quality ratings).

**Impact:** Half-life values should be treated as relative indicators, not absolute measures.

### Incomplete Transfer Records
If intermediate transfers between departments are missing from the dataset, the model will underestimate Ownership Entropy (fewer departments appear involved) and overestimate Accountability Half-Life (fewer transfers = slower apparent decay).

**Mitigation:** Ensure complaint transfer logs are complete before running analysis.

---

## 2. Model Calibration

### Scaling Factor
The decay constant uses a default scaling factor of 0.5. This value has not been calibrated against real-world resolution outcomes.

**Impact:** Absolute half-life values will shift depending on calibration. The relative ranking of complaints by half-life is more reliable than absolute day counts.

**Recommendation:** Calibrate the scaling factor against a labelled dataset of resolved vs unresolved complaints before using outputs for policy decisions.

### Counterfactual Simulator Weights
The resolution improvement formula uses fixed weights (0.4 and 0.3) that have not been empirically derived. These are reasonable starting assumptions but should be validated with domain experts.

---

## 3. Scope Boundaries

### Municipal Context Only
This system is designed specifically for municipal complaint workflows. It has not been tested on state or national government systems, private sector workflows, or emergency response systems.

### Historical Analysis Only
Mandate Vacuum analyzes historical transfer data only. It does not track complaints in real time, predict future complaint routing, or monitor individual department or worker performance.

### No Causal Claims
The system identifies correlations and structural patterns. It does not establish causality. High entropy correlating with longer resolution times is a valid finding — claiming it causes longer resolution times requires additional causal analysis beyond this system's scope.

---

## 4. AI-Generated Insights (Gemini API)

Narrative summaries generated by the Google Gemini API are synthesized interpretations of metric outputs, not independent analyses. They are subject to hallucination or oversimplification in edge cases.

**Recommendation:** All Gemini-generated policy narratives should be reviewed by a domain expert before use in formal policy documents.

---

## 5. Sample Data

The sample_data/sample_complaints.csv file is synthetically generated for demonstration purposes. It does not represent any real municipality's data and should not be used to draw real-world policy conclusions.

---

## 6. What This System Is Not

| This system does NOT...          | Why it matters to state this                  |
|----------------------------------|-----------------------------------------------|
| Monitor individual workers       | Prevents misuse for performance surveillance  |
| Track complaints in real time    | Clarifies it is not a helpdesk tool           |
| Guarantee resolution improvement | Simulations are projections, not guarantees   |
| Replace expert policy judgment   | Outputs are decision-support, not decisions   |
| Work without clean data          | Data quality directly determines output quality |
